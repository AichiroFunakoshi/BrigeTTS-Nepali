## [2.9.3] - 2026-07-08

### Changed
- **既定の翻訳方式を「順送り（石畳方式）」に反転**（Issue #73 第1段階）。実機A/B比較で順送りが「レスポンス・表示安定・伝達力」のすべてで標準方式を上回るとの評価を受けたもの。標準（全文再翻訳）は設定に残し、不具合時のフォールバックとする。**保存済みの翻訳方式設定は尊重**され、未設定の端末にのみ新既定が適用される。設定UIから「β」表記を外し、説明文を更新
- 第2段階（標準方式コードの撤廃判断）は1〜2リリースの安定運用後に判断（#73）。#72（暫定訳の並走表示）は別途設計

## [2.9.2] - 2026-07-08

### Added
- ユーザー辞書に「誤認識される表記」（誤認識パターン）を追加。音声認識が固有名詞を読みの同じ別語に書き起こす場合（例:「禄寿園」→「60円」）、認識テキストの段階で正しい表記へ決定的に置換する（LLM依存の復元では原理的に直せないケースへの対処。表示・翻訳・順送りチャンクのすべてに適用）。プロンプトの辞書セクションにも誤認識例として明示
- 設定に「日本語の読み上げ音声」の選択を追加（既定は従来どおり自動＝高品質優先）。棒読みに感じる場合に端末内の別の声へ切り替えられる
- 異常終了の検出: 録音中のフリーズ→強制終了はJSエラーを出さずエラーレポートに残らなかったため、録音状態と直近ログをハートビートで記録し、次回起動時に異常終了を検知して直前ログ付きでレポート可能にした

### Fixed
- iOSネイティブでTTSの最初の語（"the" 等）が発声されないことがある問題: オーディオセッション切替完了通知の直後は出力経路が安定しないため、150ms待ってから発話を開始するようにした

## [2.9.1] - 2026-07-08

### Added
- 翻訳方式「順送り（β）」を設定に追加（既定は従来の「標準（全文再翻訳）」のままで挙動不変・A/B比較用）。発話の確定セグメントごとに前から訳を積み上げる凍結方式（確定訳は組み変わらない）で、録音停止時に整形パスが原文全体と突き合わせて誤訳・訳抜け・冗長を修正する。整形は順送りの語順骨格（短文・主要部先出し）を保持する方針（根拠: 順送り訳は逆送り訳より聞き手の客観的理解度が高い傾向 — 児島ら「順送り訳における聞き手の主観的・客観的理解度の比較分析」NLP2026 C8-15。「自然な語順」への再編は流暢性の錯覚を招き実理解を下げ得る）。設計は `docs/POST_MEASUREMENT_PLAN.md` §3（増分翻訳）を土台にした
- `TranslatorService.createPayload` / `translateStream` に `userContent`（userメッセージ差し替え）を追加。未指定時は従来と同一
- `PromptService.getRefinementSystemPrompt`（整形パス用プロンプト。会話領域・ユーザー辞書の注入は翻訳プロンプトと共通）
- 評価ハーネスに `--monotonic` オプションと文脈維持5ケース `eval/cases-monotonic.json` を追加（チャンク翻訳の代名詞・省略主語の維持を検証。既存 `cases.json` 50件は不変）

### Fixed
- 音声認識の自動再起動後、直前セッションと同じ位置・同じ内容の発話（短い相槌の繰り返し等）が処理済みと誤判定されて失われる問題（resultIdに認識セッション連番を付与）

### Docs

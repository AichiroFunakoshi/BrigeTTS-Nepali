# 手動検証チェックリスト

作成日: 2026-05-18

このチェックリストは、`app.js` の責務分割や UI 改修の前後で既存機能が壊れていないか確認するための基準です。実機確認は iPhone Safari を最優先にし、可能なら Android Chrome でも確認します。

## 1. 起動と PWA 基本動作

- [ ] GitHub Pages またはローカル HTTPS で `index.html` が開く
- [ ] 初回起動時に API キー設定モーダルが表示される
- [ ] API キー説明に「リポジトリには保存されない」旨が表示される
- [ ] `manifest.json` が読み込まれ、ホーム画面追加候補として認識される
- [ ] 2回目以降の起動でアプリシェルが素早く表示される
- [ ] オフライン時に「翻訳には通信が必要」という案内が表示される

## 2. API キーと設定

- [ ] API キーを入力して「保存して開始」を押すとモーダルが閉じる
- [ ] 再読み込み後も API キー入力済み状態で起動する
- [ ] 「APIキーリセット」でキーが削除され、再読み込み後にモーダルが再表示される
- [ ] TTS ON/OFF 設定が再読み込み後も保持される
- [ ] 読み上げ速度が再読み込み後も保持される
- [ ] 文字サイズが再読み込み後も保持される

## 3. 音声認識

- [ ] 「開始」を押すと日本語入力モードで録音が始まる
- [ ] 「Start」を押すと英語入力モードで録音が始まる
- [ ] マイク許可ダイアログが出た場合、許可後に録音状態へ進む
- [ ] 原文欄に認識テキストが表示される
- [ ] 「停止」で録音が止まる
- [ ] 録音停止後にボタン状態が待機状態へ戻る

## 4. 翻訳

- [ ] 日本語入力時、翻訳欄に英語が表示される
- [ ] 英語入力時、翻訳欄に日本語が表示される
- [ ] 翻訳中は状態表示が「翻訳中」になる
- [ ] 翻訳完了後、翻訳ボックスが再生可能状態になる
- [ ] 通信エラー時に画面上へエラーメッセージが表示される
- [ ] API クォータ不足時に分かりやすいエラーが表示される

## 5. TTS

- [ ] 翻訳ボックスをタップすると翻訳結果が読み上げられる
- [ ] 読み上げ中に再タップすると停止する
- [ ] 日本語から英語の翻訳では英語音声が選ばれる
- [ ] 英語から日本語の翻訳では日本語音声が選ばれる
- [ ] TTS OFF の時は読み上げが発火しない
- [ ] 録音停止またはリセット時に読み上げも停止する

## 6. デバウンス最適化

- [ ] 設定画面に日本語・英語の現在値が表示される
- [ ] 使用中にサンプル数が増える
- [ ] サンプル不足時は「最適化を実行」が無効
- [ ] 必要サンプル数到達後に最適化ボタンが有効になる
- [ ] 「設定をリセット」で値だけが初期値へ戻る
- [ ] 「完全リセット」で収集データも削除される

## 7. レイアウト

- [ ] 縦画面で原文と翻訳が上下に表示される
- [ ] 横画面で原文と翻訳が左右に表示される
- [ ] 小さい画面で主要ボタンが画面外にはみ出さない
- [ ] 設定モーダル内のボタンと説明文が重ならない
- [ ] 文字サイズ「特大」でも原文・翻訳欄が破綻しない

## 8. エラーレポート

- [ ] エラー発生時に「エラーを報告」ボタンが表示される
- [ ] レポートモーダルに環境情報とログが表示される
- [ ] GitHub 報告ボタンが `Bridge-TTS-Codex-` の Issue 作成画面へ遷移する
- [ ] レポート内に API キーそのものが含まれない

## リファクタリング時の合格条件

- `node --check app.js` が通る
- `node --check sw.js` が通る
- ローカル配信で初期画面が表示される
- 上記チェックリストのうち、変更箇所に関係する項目が通る
- API キーや会話内容をリポジトリに追加しない

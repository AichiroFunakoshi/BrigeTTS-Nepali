1. はじめに — このアプリでできること

BrigeTTS(Nepali)は、日本語とネパール語のあいだをリアルタイムで音声翻訳するアプリです。 マイクに向かって話すだけで文字起こし・翻訳が行われ、翻訳結果をタップすると音声で読み上げてくれます。 通訳がいない場面での会話や、ネパール語でのやりとりのブリッジとして活用できます。

できること（要点）
  • 日本語で話す → ネパール語に翻訳・表示・読み上げ
  • ネパール語で話す → 日本語に翻訳・表示・読み上げ
  • 翻訳はリアルタイムのストリーミング表示
  • 翻訳ボックスをタップするだけで音声再生（TTS）
  • 直近20件の会話履歴を自動保存
できないこと・制限
  • 日本語・ネパール語ペアのみ対応（他の言語ペアは非対応）
  • 翻訳にはインターネット接続が必須（オフライン翻訳不可）
  • 翻訳にはOpenAI APIキーの登録が必要（有料API）
  • サイレントモード時はTTSが鳴らないことがある

2. しくみ

図2 ： PWA版と iOSアプリ版の構成比較

PWA版はSafariのWeb Speech APIで音声認識を行いますが、iOSのWKWebView（アプリ版で使うWebブラウザコンポーネント）にはその機能がありません。 そこで、SpeechRecognitionBridgeという Swift製モジュールが音声を聞き取り、 native-speech.jsというスクリプトを通じてWebアプリに渡します。 翻訳・TTS部分はPWA版と同じWebコードをそのまま利用しています。

4. 画面の見かたと基本操作

図5 ： メイン画面のUIガイド

基本操作の手順

  1. アプリを起動し、設定（歯車）からOpenAI APIキーを入力する（初回のみ）
  2. 「日本語」（赤）または「नेपाली」（緑）ボタンをタップして録音を開始する
  3. マイクに向かってはっきり話す（原文ボックスに文字が表示される）
  4. 話し終わると自動で翻訳が始まり、翻訳ボックスにリアルタイムで結果が表示される
  5. 翻訳完了後、翻訳ボックスが緑色になったらタップして音声読み上げ（再生中タップで停止）
  6. 「停止」ボタンをタップして録音を終了する
  7. オレンジのリセットボタンで入力欄をクリアできる（履歴は消えない）

5. 設定ガイド

右上の歯車アイコンをタップすると設定モーダルが開きます。

設定項目 選択肢 / 入力値 説明
OpenAI APIキー sk-...の形式で入力 翻訳に必須。端末のlocalStorageに保存され、OpenAIに直接送信される。共有端末では使用後リセット推奨
テーマ 自動 ／ ライト ／ ダーク 「自動」は端末の設定に追従。好みや視認性に合わせて選択
TTS（音声読み上げ） ON ／ OFF OFFにすると翻訳ボックスをタップしても読み上げされない
読み上げ速度 ×0.8 ／ ×0.9 ／ ×1.0 ／ ×1.1 ／ ×1.2 ゆっくり読んでほしい場合は×0.8、素早く聞きたい場合は×1.2を選択
文字サイズ 小 ／ 中 ／ 大 ／ 特大 翻訳結果などの表示文字サイズを調整。視力に合わせて設定
デバウンス最適化 使用パターン学習（任意） 合計50件・各言語15件の使用データで最適化可能。初期値は日本語346ms/ネパール語300ms
APIキーについて：入力したAPIキーはサーバーには送られず、あなたの端末のブラウザ内（localStorage）にのみ保存されます。翻訳時にあなたの端末から直接OpenAIへ送信されます。

6. 会話履歴の使いかた

アプリは自動的に直近20件の会話（原文と翻訳のペア）を端末内に保存します。アプリを閉じても履歴は保持されます。

操作方法備考
履歴の確認 画面下部の会話履歴エリアを見る 各エントリに方向（日→英 / 英→日）と時刻が表示される
再生 各エントリの再生（▶）ボタンをタップ 翻訳テキストをTTSで読み上げ
コピー 各エントリの「コピー」ボタンをタップ 翻訳テキストをクリップボードにコピー
全削除 「消去」ボタンをタップ 全エントリが削除される（元に戻せない）
注意：リセットボタン（オレンジの円形矢印）をタップしても会話履歴は消えません。「消去」ボタンのみが履歴を削除します。

7. 日々の運用

図7 ： 7日署名サイクルの運用タイムライン

7日Refreshの手順

  1. PCのAltServerを起動し、iPhoneと同じWi-Fiに接続する
  2. iPhoneでAltStoreを開く
  3. 「My Apps」タブでBrigeTTS(Nepali)の「Refresh」をタップ
  4. 署名が更新され、さらに7日間使用可能になる
バックグラウンド自動更新：AltServerが起動しているPCと同一Wi-Fiに接続していると、AltStoreがバックグラウンドで自動的にRefreshしてくれることがあります。ただし必ず自動更新されるとは限らないため、定期的に手動確認を推奨します。

3アプリ制限について

無料のApple IDでサイドロードできるアプリは同時に3つまでです。すでに3つのアプリをサイドロードしている場合は、1つ削除してからBrigeTTS(Nepali)をインストールしてください。有料のApple Developer Program（年額制）に加入すると、この制限と7日の署名期限が大幅に緩和されます。

APIキー管理

APIキーは端末のlocalStorageのみに保存されます。共有端末で使う場合は、使用後に設定からAPIキーをリセットしてください。 アプリをアンインストールするとlocalStorageも削除されます。

9. 困ったときは

症状主な原因対処法
翻訳が始まらない APIキーが未設定 / インターネット未接続 設定（歯車）でAPIキーを確認。Wi-Fi / モバイル通信の接続を確認
音声が認識されない マイク権限が未許可 / 周囲の騒音 設定アプリ → BrigeTTS(Nepali) → マイクをON。静かな場所ではっきり話す
TTSが鳴らない サイレントモード / TTSがOFF / まだ翻訳中 着信音スイッチをON。設定でTTSをON確認。翻訳ボックスが緑になってからタップ
iOSアプリが起動しない 7日間の署名が失効 AltStoreで「Refresh」を実行（AltServerと同一Wi-Fiが必要）
AltStoreインストールでエラー 3アプリ制限に達している AltStoreのMy Appsから不要なアプリを削除してから再試行
翻訳結果が不自然 発話が不明瞭 / 方言・専門用語 ゆっくりはっきり話す。文が長い場合は分けて話す
PWAがホーム画面に追加できない HTTP接続 / Safariで開いていない 必ずSafariでHTTPS URLを開いてから共有 → ホーム画面に追加
APIエラーが頻発する APIキーが無効 / 残高不足 platform.openai.com でキーの有効性と残高を確認。必要に応じてチャージ
エラーレポート機能：解決できない問題が発生した場合は、画面右下の「エラーを報告」ボタンをタップすると、ログ情報を含むGitHub Issue作成画面へ移動します。

10. PWA版とアプリ版 — どちらを使う？

比較項目PWA版（Safari）iOSアプリ版（AltStore）
インストールの手軽さ ◎ Safariで開くだけ △ PCとAltServerが必要
音声認識 ○ Web Speech API ◎ iOSネイティブSFSpeechRecognizer
マイク権限 △ サイトごとに確認されることがある ◎ アプリ権限として一度許可すればOK
アプリシェルの起動 ○ Service Worker（要事前訪問） ◎ 常にローカルバンドル
定期的なメンテナンス ◎ 不要（Webが更新されれば自動） △ 7日ごとにRefreshが必要
アプリ数制限 ◎ 制限なし △ 無料Apple IDで3アプリまで
Android対応 ◎ Chrome対応 ✕ iOS専用
おすすめシーン 手軽に試したい、Android利用者 音声認識を安定させたい、iPhone常用者
まずはPWA版を試してみることをお勧めします。 SafariでGitHub Pagesのアプリを開き、共有ボタンから「ホーム画面に追加」するだけですぐに使えます。 音声認識の動作に不満がある場合や、iPhone専用でしっかり使いたい場合にiOSアプリ版を検討してください。

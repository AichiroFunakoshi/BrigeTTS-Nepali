# 翻訳評価ケース

Bridge-TTS Codex の翻訳プロンプト、モデル、レスポンス設定を変更するときに、同じ入力で品質を比較するための評価ケース集。

> **自動評価ハーネス（F11）**: 本文書のケースを拡張した機械判定可能な50ケースが `eval/cases.json` にあり、
> `OPENAI_API_KEY=... node eval/run-eval.js` で自動評価できる（費用: 1回あたり数円未満）。
> プロンプト・モデル・温度を変更する際は、変更前後で実行して合格数とcritical失敗数を比較すること。
> 本文書は評価観点（ルーブリック）と人手での確認手順の記録として維持する。

## 使い方

1. 変更前後で同じケースを入力する。
2. 翻訳結果を「意味の正確さ」「自然さ」「現場での使いやすさ」で見る。
3. 医療・接客など誤訳リスクが高いケースは、自然さより正確さを優先する。
4. 比較結果は下の「比較メモ」に追記する。

## 評価観点

- 意味の正確さ: 主語、否定、時制、数量、条件が落ちていないか。
- 自然さ: 相手にそのまま見せたり読み上げたりして不自然でないか。
- 丁寧さ: 医療・接客・依頼の場面で失礼になっていないか。
- 簡潔さ: 会話中に読むには長すぎないか。
- 安全性: 症状、薬、時間、金額、場所などを誤解させないか。
